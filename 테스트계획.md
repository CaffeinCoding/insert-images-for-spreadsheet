# Import Images Pattern - 테스트 계획

## 📋 테스트 전략 개요

### 목표

프로젝트 명세의 모든 기능이 정확하게 작동하는지 검증하고, 엣지 케이스 및 에러 상황을 철저히 테스트하여 안정적인 배포 보장

### 테스트 범위

- ✅ 기능 테스트 (Functional Testing)
- ✅ 통합 테스트 (Integration Testing)
- ✅ 성능 테스트 (Performance Testing)
- ✅ 호환성 테스트 (Compatibility Testing)
- ✅ 에러 처리 테스트 (Error Handling Testing)
- ✅ UI/UX 테스트 (UI/UX Testing)

### 테스트 우선순위

1. 🔴 **높음** (필수): 기본 배치 로직, 프리뷰, 에러 처리
2. 🟠 **중간**: 옵션 기능, 상태 관리, 성능
3. 🟡 **낮음**: UI 세부사항, 호환성

## 🛠️ 테스트 환경 설정

### 필요한 도구 및 환경

#### 개발 환경

```
- Google Apps Script 프로젝트
- Google Sheets 스프레드시트 (테스트용)
- 웹 브라우저 (Chrome, Firefox, Edge)
- 텍스트 에디터 (VS Code 권장)
- 개발자 도구 (DevTools)
```

#### 테스트 데이터

```
- 테스트 이미지 (다양한 크기, 형식)
  - 소형 (100x100px)
  - 중형 (500x500px)
  - 대형 (2000x2000px)
  - 초대형 (5000x5000px)
- 테스트 스프레드시트
  - 빈 스프레드시트
  - 데이터가 있는 스프레드시트
  - 병합 셀이 있는 스프레드시트
```

#### 테스트 스프레드시트 준비

```
1. 테스트용 스프레드시트 생성
   - 이름: "Import Images Pattern - Test"

2. 테스트 시트 구성
   - Sheet1: 기본 테스트
   - Sheet2: 간격 테스트
   - Sheet3: 비활성 셀 테스트
   - Sheet4: 크기 조절 테스트
   - Sheet5: 병합 셀 테스트
   - Sheet6: 에러 테스트
```

#### 테스트 이미지 준비

```
images/
├── portrait_small.jpg      (100x200)
├── landscape_small.jpg     (200x100)
├── square_medium.jpg       (500x500)
├── portrait_large.jpg      (1000x2000)
├── landscape_large.jpg     (2000x1000)
└── corrupted.jpg           (손상된 파일)
```

## 📝 테스트 케이스 정의

### 1️⃣ 기본 이미지 배치 테스트 (TC-1 ~ TC-5)

#### TC-1: 단순 2x2 배치

```
조건:
- 이미지: 4개 (명명: img1.jpg, img2.jpg, img3.jpg, img4.jpg)
- 선택 셀: A1
- 패턴: 행 2, 열 2
- 간격: 행 0, 열 0
- 비활성 셀: 없음

예상 결과:
- img1 → A1
- img2 → B1
- img3 → A2
- img4 → B2

검증 방법:
1. Sidebar에서 이미지 4개 추가
2. 행 2, 열 2 설정
3. 프리뷰 확인 (4개 셀이 녹색 표시)
4. 완료 버튼 클릭
5. 각 셀에 이미지 정확히 배치 확인
```

#### TC-2: 3x3 배치

```
조건:
- 이미지: 9개
- 선택 셀: B2
- 패턴: 행 3, 열 3
- 간격: 행 0, 열 0

예상 결과:
- 9개 이미지가 B2부터 D4 범위에 배치

검증 방법:
1. 이미지 9개 추가
2. B2 셀 선택
3. 행 3, 열 3 설정
4. 완료 후 배치 확인
```

#### TC-3: 단일 행 배치 (1x5)

```
조건:
- 이미지: 5개
- 선택 셀: A1
- 패턴: 행 1, 열 5

예상 결과:
- A1, B1, C1, D1, E1에 배치

검증 방법:
1. 이미지 5개 추가
2. 행 1, 열 5 설정
3. 한 행으로 배치 확인
```

#### TC-4: 단일 열 배치 (5x1)

```
조건:
- 이미지: 5개
- 선택 셀: A1
- 패턴: 행 5, 열 1

예상 결과:
- A1, A2, A3, A4, A5에 배치

검증 방법:
1. 이미지 5개 추가
2. 행 5, 열 1 설정
3. 한 열로 배치 확인
```

#### TC-5: 최대 범위 배치 (50x50)

```
조건:
- 이미지: 100개
- 선택 셀: A1
- 패턴: 행 50, 열 50 (2500 셀)

예상 결과:
- 100개만 배치, 나머지 셀은 비움

검증 방법:
1. 이미지 100개 추가
2. 행 50, 열 50 설정
3. 성능 문제 없이 배치 확인
```

### 2️⃣ 간격 설정 테스트 (TC-6 ~ TC-10)

#### TC-6: 행 간격 1 적용

```
조건:
- 이미지: 2개
- 선택 셀: A1
- 패턴: 행 2, 열 1
- 행 간격: 1
- 열 간격: 0

예상 결과:
- img1 → A1
- img2 → A3 (1행 간격)
- A2는 빔

검증:
| A | B | C |
|---|---|---|
| ✓img1 | | |
|   |   |   |
| ✓img2 | | |

세부 검증:
1. 프리뷰에서 A1, A3만 녹색 표시
2. A2는 비어있음
3. 완료 후 실제 배치 확인
```

#### TC-7: 열 간격 1 적용

```
조건:
- 이미지: 2개
- 선택 셀: A1
- 패턴: 행 1, 열 2
- 행 간격: 0
- 열 간격: 1

예상 결과:
- img1 → A1
- img2 → C1 (1열 간격)

검증:
| A | B | C |
|---|---|---|
| ✓img1 | | ✓img2 |
```

#### TC-8: 행 간격 2, 열 간격 1 적용

```
조건:
- 이미지: 4개
- 선택 셀: B1
- 패턴: 행 2, 열 2
- 행 간격: 1
- 열 간격: 1

예상 결과:
- B1(0,0), D1(0,3), B3(2,1), D3(2,3)

검증:
| A | B | C | D | E |
|---|---|---|---|---|
|   |✓img1|   |✓img2|   |
|   |     |   |     |   |
|   |✓img3|   |✓img4|   |
```

#### TC-9: 최대 간격 테스트

```
조건:
- 이미지: 2개
- 선택 셀: A1
- 패턴: 행 10, 열 10
- 행 간격: 9
- 열 간격: 9

예상 결과:
- img1 → A1
- img2 → K10 (스프레드시트 범위 내)
```

#### TC-10: 간격이 패턴을 벗어나는 경우

```
조건:
- 이미지: 3개
- 선택 셀: A1
- 패턴: 행 2, 열 2
- 행 간격: 1 (계산: 행 1 + 간격 1 = 행 3으로 패턴 벗어남)
- 열 간격: 0

예상 결과:
- img1 → A1
- img2 → A3 (패턴 벗어남)
- img3 → 배치 불가

검증:
- 경고 메시지 또는 무시 처리 확인
```

### 3️⃣ 비활성 셀 테스트 (TC-11 ~ TC-15)

#### TC-11: 단일 비활성 셀

```
조건:
- 이미지: 4개
- 선택 셀: A1
- 패턴: 행 2, 열 2
- 비활성 셀: (1, 2) - 1행 2열

예상 결과:
- (1,1) → img1
- (1,2) → 비활성 (건너뜀)
- (2,1) → img2
- (2,2) → img3

배치 결과:
| A | B |
|---|---|
|✓img1| X |
|✓img2|✓img3|

검증:
1. 비활성 셀 표에서 (1,2) 셀 선택
2. 프리뷰에서 해당 셀 회색 표시
3. 실제 배치 확인
```

#### TC-12: 다중 비활성 셀

```
조건:
- 이미지: 6개
- 선택 셀: A1
- 패턴: 행 3, 열 3 (9 셀)
- 비활성 셀: (1,2), (2,2), (3,1) - 3개

예상 결과:
- 6개만 배치 (9 - 3 = 6)
- 해당 셀들 건너뜀

배치 결과:
| A | B | C |
|---|---|---|
|✓img1| X |✓img2|
|✓img3| X |✓img4|
| X |✓img5|✓img6|
```

#### TC-13: 비활성 셀이 모두 차는 경우

```
조건:
- 이미지: 5개
- 선택 셀: A1
- 패턴: 행 2, 열 2 (4 셀)
- 비활성 셀: (1,1), (1,2), (2,1), (2,2) - 모두

예상 결과:
- 배치할 수 없음 (모든 셀 비활성)
- 5개 이미지 모두 무시
- 경고 메시지
```

#### TC-14: 비활성 셀 토글

```
조건:
- 패턴: 행 2, 열 2
- 초기 비활성 셀: (1,2)

동작:
1. (1,2) 셀 클릭 → 활성화
2. (2,1) 셀 클릭 → 비활성화
3. (1,2) 셀 다시 클릭 → 비활성화

예상 결과:
- 토글 방식으로 상태 변경
- 프리뷰 실시간 반영
```

#### TC-15: 행/열 개수 변경 시 비활성 셀 초기화

```
조건:
- 초기 비활성 셀: 3x3 패턴에서 (2,3)

동작:
1. 행 개수 변경: 3 → 4

예상 결과:
- 비활성 셀 초기화 (4x3 패턴이 되므로)
- 프리뷰에서 비활성 셀 표 다시 생성
```

### 4️⃣ 이미지 개수 처리 테스트 (TC-16 ~ TC-20)

#### TC-16: 이미지 부족

```
조건:
- 이미지: 2개
- 패턴: 행 2, 열 2 (4 셀 필요)

예상 결과:
- 2개만 배치
- 남은 2개 셀은 비움

검증:
| A | B |
|---|---|
|✓img1|✓img2|
|   |   |

프리뷰: 4개 셀 모두 녹색이지만 2개만 이미지 표시
```

#### TC-17: 이미지 초과

```
조건:
- 이미지: 10개
- 패턴: 행 2, 열 2 (4 셀)

예상 결과:
- 4개만 배치
- 6개 이미지 무시 (경고 없음)
```

#### TC-18: 이미지 1개

```
조건:
- 이미지: 1개
- 패턴: 행 3, 열 3

예상 결과:
- (1,1)에만 배치
- 나머지 8개 셀 비움
```

#### TC-19: 이미지 0개 (에러)

```
조건:
- 이미지: 0개

예상 결과:
- 완료 버튼 비활성화
- "가져온 이미지가 없을 경우 비활성화" 메시지
```

#### TC-20: 비활성 셀 때문에 배치 불가능

```
조건:
- 이미지: 5개
- 패턴: 행 2, 열 2 (4 셀)
- 비활성 셀: 3개

실제 배치 가능: 4 - 3 = 1개

예상 결과:
- 1개만 배치
- 4개 이미지 무시
```

### 5️⃣ 이미지 크기 조절 테스트 (TC-21 ~ TC-30)

#### TC-21: 기본 설정 (셀 크기에 맞춤)

```
조건:
- 이미지: 500x500px
- 설정: 셀 크기에 맞춤 ON (기본)
- 선택 셀: A1
- 셀 크기: 높이 100px, 너비 100px

예상 결과:
- 이미지 크기: 100px x 100px
- 비율 유지 안 함

검증:
1. Sidebar에서 크기 확인
2. 실제 배치 후 이미지 크기 확인
```

#### TC-22: 비율 유지, 가로만 지정

```
조건:
- 이미지: 500x1000px (비율 1:2)
- 설정: 비율 유지 ON
- 가로 크기: 2셀 (200px)
- 세로 크기: 비워둠

예상 결과:
- 세로 크기 자동 계산: 200 × (1000/500) = 400px (4셀)

검증:
1. 가로만 입력
2. 세로 자동 계산 확인
```

#### TC-23: 비율 유지, 세로만 지정

```
조건:
- 이미지: 1000x500px (비율 2:1)
- 설정: 비율 유지 ON
- 가로 크기: 비워둠
- 세로 크기: 1셀 (100px)

예상 결과:
- 가로 크기 자동 계산: 100 × (1000/500) = 200px (2셀)

검증:
1. 세로만 입력
2. 가로 자동 계산 확인
```

#### TC-24: 비율 유지, 가로/세로 모두 지정

```
조건:
- 이미지: 500x500px
- 설정: 비율 유지 ON
- 가로 크기: 2셀
- 세로 크기: 3셀

예상 결과:
- 비율 유지 자동 OFF
- 2셀 x 3셀로 변형되어 배치

검증:
1. 가로/세로 모두 입력
2. 체크박스 자동으로 OFF 확인
```

#### TC-25: 비율 유지 OFF, 자유 크기 지정

```
조건:
- 이미지: 500x500px
- 설정: 비율 유지 OFF
- 가로: 2셀, 세로: 4셀

예상 결과:
- 2x4 크기로 변형
- 비율 무시하고 배치
```

#### TC-26: 병합 셀 선택

```
조건:
- 이미지: 500x500px
- 병합 셀 선택: A1:C3 (3x3 병합)
- 설정: 셀 크기에 맞춤 ON

예상 결과:
- 병합된 전체 크기 (3셀 x 3셀)에 맞춤
- 개별 셀 크기로 계산 안 함

검증:
1. 병합 셀 선택
2. 예상 크기 확인
3. 배치 후 이미지 크기 확인
```

#### TC-27: 셀 동적 크기 변경

```
조건:
- 이미지 배치 완료: 2x2 크기
- 이미지 삽입 후 셀 높이 변경: 50px → 100px

예상 결과:
- 이미지 크기 자동 조정 (비율 유지 설정 따라)
- 선택 셀 기준으로 크기 조정

검증:
1. 배치 완료
2. 셀 높이 변경
3. 이미지 크기 자동 조정 확인
```

#### TC-28: 최소 크기 테스트

```
조건:
- 이미지: 1x1 크기로 설정

예상 결과:
- 최소 크기 유지 (에러 없음)
```

#### TC-29: 최대 크기 테스트

```
조건:
- 이미지: 50셀 x 50셀로 설정

예상 결과:
- 최대 크기 유지 (스프레드시트 범위 내)
```

#### TC-30: 다양한 비율 이미지

```
조건:
- 초상화 이미지 (2:3 비율)
- 가로 이미지 (3:2 비율)
- 정사각형 이미지 (1:1 비율)

예상 결과:
- 각각 비율 정확히 유지
- 선택한 셀 크기에 맞춤

검증:
1. 3개 이미지 각각 테스트
2. 비율 정확도 확인
```

### 6️⃣ 셀 선택 테스트 (TC-31 ~ TC-35)

#### TC-31: 단일 셀 선택

```
조건:
- 선택: A1

예상 결과:
- Sidebar에 "A1" 표시
- 프리뷰에서 A1 파란색 표시
```

#### TC-32: 범위 선택 (지원 안 함)

```
조건:
- 선택: A1:D4 범위

- 처음 선택 셀 (A1)만 인식

```

#### TC-33: 셀 재선택 (onSelectionChange)

```
조건:
- 초기 선택: A1
- 재선택: C3

동작:
1. A1 선택
```
